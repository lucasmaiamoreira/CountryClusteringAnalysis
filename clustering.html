4 - Escolha de algoritmos.

  1. Escreva em tpicos as etapas do algoritmo de K-mdias at sua convergncia.
  • Inicializao: Escolher um nmero de clusters (k) e selecionar aleatoriamente k pontos como centrides iniciais.
  • Atribuio de Pontos: Atribuir cada ponto de dados ao cluster cujo centride o mais prximo, com base na distncia Euclidiana.
  • Recalcular os Centrides: Calcular o novo centride de cada cluster como a mdia das posies de todos os pontos que pertencem a esse cluster.
  • Repetir: Repetir os passos de atribuio de pontos e recalculo dos centrides at que os centrides no mudem mais ou o movimento entre os centrides seja mnimo (isto , o algoritmo tenha convergido).
  1. O algoritmo de K-mdias converge at encontrar os centrides que melhor descrevem os clusters encontrados (at o deslocamento entre as interaes dos centrides ser mnimo). Lembrando que o centride o baricentro do cluster em questo e no representa, em via de regra, um dado existente na base. Refaa o algoritmo apresentado na questo 1 a fim de garantir que o cluster seja representado pelo dado mais prximo ao seu baricentro em todas as iteraes do algoritmo. Obs: nesse novo algoritmo, o dado escolhido ser chamado medide.

# Funo para calcular a distncia Euclidiana
def euclidean_distance(a, b):
    return np.linalg.norm(a - b)

# Funo para encontrar o medide de um cluster
def calculate_medoid(cluster_points):
    distances = np.zeros(len(cluster_points))
    for i in range(len(cluster_points)):
        distances[i] = np.sum([euclidean_distance(cluster_points[i], other) for other in cluster_points])
    medoid_idx = np.argmin(distances)  # Escolhe o ndice do ponto mais prximo
    return cluster_points[medoid_idx]

# Aplicando a modificao para usar Medides
def k_medoids_algorithm(data, k, max_iters=100):
    # Inicializao: Escolher k pontos aleatrios como medides
    medoids = data[np.random.choice(data.shape[0], k, replace=False)]
    labels = np.zeros(data.shape[0])

    for _ in range(max_iters):
        # Atribuir pontos ao cluster mais prximo do medide
        for i in range(data.shape[0]):
            distances = [euclidean_distance(data[i], medoid) for medoid in medoids]
            labels[i] = np.argmin(distances)

        # Recalcular os medides
        new_medoids = np.copy(medoids)
        for i in range(k):
            cluster_points = data[labels == i]
            new_medoids[i] = calculate_medoid(cluster_points)
        
        # Se os medides no mudaram, o algoritmo convergiu
        if np.array_equal(medoids, new_medoids):
            break
        medoids = np.copy(new_medoids)

    return labels, medoids

  1. O Algoritmo de K-Mdias Sensvel a Outliers nos Dados. Explique:
  • O algoritmo de K-mdias sensvel a outliers porque o centride de cada cluster calculado como a mdia dos pontos. Se houver outliers (pontos distantes), eles podem distorcer essa mdia e fazer o centride se mover para uma posio que no represente bem o restante dos dados.
  • Isso pode levar a uma atribuio incorreta de pontos aos clusters, j que o centride pode ser influenciado por outliers e no representar corretamente o grupo de dados.
  1. Por Que o Algoritmo de DBScan Mais Robusto Presena de Outliers?:
  • O DBScan mais robusto a outliers porque ele no usa mdias para formar os clusters. Em vez disso, ele se baseia na densidade dos pontos.
  • DBScan identifica clusters como regies de alta densidade de pontos prximos uns aos outros.
  • Outliers, que so pontos isolados ou em regies de baixa densidade, so identificados como rudo e no so includos em nenhum cluster.
  • Dessa forma, os outliers no afetam a formao dos clusters, o que torna o DBScan mais adequado para conjuntos de dados com outliers.
