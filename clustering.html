4 - Escolha de algoritmos.¶

  1. Escreva em tópicos as etapas do algoritmo de K-médias até sua convergência.
  • Inicialização: Escolher um número de clusters (k) e selecionar aleatoriamente k pontos como centróides iniciais.
  • Atribuição de Pontos: Atribuir cada ponto de dados ao cluster cujo centróide é o mais próximo, com base na distância Euclidiana.
  • Recalcular os Centróides: Calcular o novo centróide de cada cluster como a média das posições de todos os pontos que pertencem a esse cluster.
  • Repetir: Repetir os passos de atribuição de pontos e recalculo dos centróides até que os centróides não mudem mais ou o movimento entre os centróides seja mínimo (isto é, o algoritmo tenha convergido).
  1. O algoritmo de K-médias converge até encontrar os centróides que melhor descrevem os clusters encontrados (até o deslocamento entre as interações dos centróides ser mínimo). Lembrando que o centróide é o baricentro do cluster em questão e não representa, em via de regra, um dado existente na base. Refaça o algoritmo apresentado na questão 1 a fim de garantir que o cluster seja representado pelo dado mais próximo ao seu baricentro em todas as iterações do algoritmo. Obs: nesse novo algoritmo, o dado escolhido será chamado medóide.

# Função para calcular a distância Euclidiana
def euclidean_distance(a, b):
    return np.linalg.norm(a - b)

# Função para encontrar o medóide de um cluster
def calculate_medoid(cluster_points):
    distances = np.zeros(len(cluster_points))
    for i in range(len(cluster_points)):
        distances[i] = np.sum([euclidean_distance(cluster_points[i], other) for other in cluster_points])
    medoid_idx = np.argmin(distances)  # Escolhe o índice do ponto mais próximo
    return cluster_points[medoid_idx]

# Aplicando a modificação para usar Medóides
def k_medoids_algorithm(data, k, max_iters=100):
    # Inicialização: Escolher k pontos aleatórios como medóides
    medoids = data[np.random.choice(data.shape[0], k, replace=False)]
    labels = np.zeros(data.shape[0])

    for _ in range(max_iters):
        # Atribuir pontos ao cluster mais próximo do medóide
        for i in range(data.shape[0]):
            distances = [euclidean_distance(data[i], medoid) for medoid in medoids]
            labels[i] = np.argmin(distances)

        # Recalcular os medóides
        new_medoids = np.copy(medoids)
        for i in range(k):
            cluster_points = data[labels == i]
            new_medoids[i] = calculate_medoid(cluster_points)
        
        # Se os medóides não mudaram, o algoritmo convergiu
        if np.array_equal(medoids, new_medoids):
            break
        medoids = np.copy(new_medoids)

    return labels, medoids

  1. O Algoritmo de K-Médias é Sensível a Outliers nos Dados. Explique:
  • O algoritmo de K-médias é sensível a outliers porque o centróide de cada cluster é calculado como a média dos pontos. Se houver outliers (pontos distantes), eles podem distorcer essa média e fazer o centróide se mover para uma posição que não represente bem o restante dos dados.
  • Isso pode levar a uma atribuição incorreta de pontos aos clusters, já que o centróide pode ser influenciado por outliers e não representar corretamente o grupo de dados.
  1. Por Que o Algoritmo de DBScan é Mais Robusto à Presença de Outliers?:
  • O DBScan é mais robusto a outliers porque ele não usa médias para formar os clusters. Em vez disso, ele se baseia na densidade dos pontos.
  • DBScan identifica clusters como regiões de alta densidade de pontos próximos uns aos outros.
  • Outliers, que são pontos isolados ou em regiões de baixa densidade, são identificados como ruído e não são incluídos em nenhum cluster.
  • Dessa forma, os outliers não afetam a formação dos clusters, o que torna o DBScan mais adequado para conjuntos de dados com outliers.
